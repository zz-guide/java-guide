# 设计模式

    开闭原则：对扩展开放、对修改关闭。
    单一指责原则：一个类只做一件事。
    依赖倒转原则：类似于ioc，采用接口编程。
    迪米特原则：高内聚，低耦合。
    接口隔离原则：应该使用多个接口，而不是用单一的总接口。
    合成复用原则：尽量使用对象组合，而不是继承来达到复用目的。
    里氏替换原则：子类可以扩展父类的功能，但不能改变原有的功能。
